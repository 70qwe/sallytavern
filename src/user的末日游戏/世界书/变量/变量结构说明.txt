本 schema 面向《user 的末日游戏》MVU 变量，依据《变量.txt》九大模块拆解而来，并与酒馆助手 MVU / JSON Patch 增量更新兼容。

世界：使用字符串承载叙事日期与时刻（合并写入 `世界.日期`，格式 `XX年X月X日 XX：XX`）与地点天气；生存天数与季节分工表达「第几天」与自然季节；「当前赛季」「赛季预计持续时间」单独描述玩法赛季（如极寒季），避免与自然季节混淆。

主角：种族默认人类，可被叙事改写；固定天赋「梦魇欲魔」及阶段一至四，附四档「能力列表」文案；UID 预留占位可在渲染层替换为「姓名首字母-00010086」格式；登陆昵称默认可为「[未命名]」；SP、污染值、饱食度允许高于传统百分比上限以呈现失控或超载叙事；力量/体质/敏捷采用「当前/上限」嵌套对象——initvar 不写死「当前」，首次开局盘点时据 {{user}} 人设与体型档位区间写入（详见变量更新规则）；角色总评采用 E~SSS 稀有度枚举，并以「角色总评评语」存放吐槽式短评；「衣着」九槽（上衣/下装/外套/帽子/内衣裤/袜子/鞋子/饰品/手持物）initvar 为空；首次开局盘点或衣着仍全空时，变量 Patch 须据 {{user}} 与环境写入默认着装（详见变量更新规则）；正文写穿脱换损时同步更新对应槽。

物品栏、论坛帖子、交易挂单等均使用 record，键为随机 id，便于 insert/remove Patch；物品栏含 描述、获取来源；交易行系统商城/玩家交易含 描述；狩猎名单拆成「进行中」与「已奴隶」，每条含昵称、种族、污染值（与主角相同：污染叙事超过 100 时可改写种族）、三维、精液质量、沦陷度、当前位置与用途、「衣着」九槽等字段；进行中条目沦陷度满 100 时必须转入「已奴隶」（不再保留沦陷度字段，衣着随条目保留）。

「下一步行动」已自变量表移除：多 API 模式下由主 API 在 `<option>` 中给出四条可选意图；单 API 模式亦以正文选项为主，不再写入 stat_data。交易行拆系统商城（每日十项基础生存物资，随生存天数脚本刷新）与玩家交易以区分每日货架与二手挂单；`交易行._商城刷新于天数` 由脚本维护。副本列表与排行榜同样使用 record，方便增量追加。

全部对象字段均提供 prefault，确保重复 parse 幂等并降低 Patch 写入失败率。
